## Style Guide (черновик): RoboMan — UI для владельцев и преподавателей

### Цели дизайна
- **Профессиональная ИС**: аккуратно, “дорого”, без визуального шума.
- **Desktop-first**: основной сценарий — работа за компьютером.
- **Быстрые сценарии**: за 2–3 клика найти нужный месяц/филиал/преподавателя.
- **Чёткое разграничение ролей**: преподаватель не видит выручку/цену; владелец видит финансовые метрики.

### Принципы интерфейса
- **Сетка**: 12‑колоночная, контент 1200–1440px, с боковыми полями 24–32px.
- **Карточки KPI**: 4–6 карточек в одну строку на desktop (адаптация 2×2 на меньших ширинах).
- **Таблицы**: фиксированный заголовок, зебра‑строки, сортировка, быстрые фильтры.
- **Фильтры**: всегда сверху (период/отдел/филиал/преподаватель). Кнопка “Сбросить”.

### Цвета (база)
Примечание: конкретные HEX можно финализировать на этапе UI-реализации, важно сохранить характер: нейтральный светлый фон + один акцент.

- **Фон приложения**: светлый нейтральный (серо‑белый).
- **Поверхности (карточки/таблицы)**: белый с мягкой границей.
- **Акцент**: один основной цвет для кнопок/ссылок/выделений.
- **Семантика**:
  - успех (зелёный), предупреждение (янтарный), ошибка (красный), инфо (синий).

### Типографика
- 1–2 шрифта максимум (например, современный гротеск).
- Заголовки: чёткая иерархия (H1/H2/H3), без “крика”.
- Цифры KPI — крупные, контрастные, с подписью.

### Компоненты (минимальный набор)
- **Sidebar навигации**: Дашборд, Занятия, Филиалы, Преподаватели, Инструкции, Настройки (владелец).
- **Topbar фильтров**: период + контекстные фильтры.
- **KPI Card**: заголовок, значение, подпись/дельта.
- **Chart Card**: график + легенда + подсказки.
- **Data Table**: сортировка, пагинация, sticky header.
- **Modal подтверждения**: удаление занятия (только владелец), предупреждения.
- **Toasts/Alerts**: успех сохранения, ошибка, предупреждение.

### Дашборд владельца (рекомендуемая компоновка)
- Верх: фильтры (период, отдел, филиал, преподаватель).
- Далее: KPI (выручка, платные, пробные, всего, среднее).
- Далее: график динамики.
- Далее: таблица занятий за период + топ‑блоки.

### Дашборд преподавателя
- Верх: период (месяц) + переключение по месяцам.
- KPI: “зарплата за период (предварительно)”, “всего занятий проведено”.
- Таблица: занятия за период (без цены и выручки).

### Пустые состояния и ошибки
- Если нет занятий за период: короткий текст + CTA “Создать занятие”.
- Ошибка сети: понятное сообщение + “Повторить”.
